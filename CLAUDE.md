# proj_checkers

AI를 활용하는 사용자가 AI를 목적에 맞게, 효과적으로 다루고 있는지 점검하기 위한 스킬 모음 프로젝트.

## 프로젝트 목적

사용자가 AI에게 내리는 지시(프롬프트)의 품질을 높이고, AI의 결과물을 객관적으로 평가할 수 있는 도구를 만든다.

구체적으로:
- 프롬프트가 명확하고 구체적으로 작성되었는지 검토·개선
- AI 결과물이 프롬프트의 요구사항을 얼마나 잘 충족했는지 정량 평가

## 디렉토리 구조

```
proj_checkers/
└── .claude/
    └── skills/          # Claude Code 스킬 파일 모음
        ├── prompt-clarify.md   # 프롬프트 명확화 스킬
        └── eval-rubric.md      # AI 결과물 평가 지표 생성 스킬
```

## 스킬 목록

### `/prompt-clarify` — 프롬프트 명확화 도우미

사용자가 작성한 프롬프트를 분석하고, 역질문을 통해 더 명확하고 구체적인 프롬프트로 개선한다.

- 최대 3라운드 역질문 진행
- 라운드당 질문 수는 스킬이 필요하다고 판단한 수에 따라 결정
- 3라운드 종료 후 또는 충분히 구체화된 시점에 개선된 프롬프트 출력

### `/eval-rubric` — 평가 지표 생성기

완성된 프롬프트를 분석해 AI 결과물을 정량적으로 평가할 수 있는 지표를 생성한다.

- 체크리스트 (Yes/No 충족 여부 확인)
- 점수표 (항목별 배점 + 과락 기준)
- AI 결과물이 함께 제공되면 즉시 평가 수행
- 과락 항목 발생 시 구체적인 피드백 제공

## 개발 가이드라인

### 스킬 추가 원칙

- 각 스킬은 `.claude/skills/` 에 독립된 `.md` 파일로 작성
- 스킬 파일은 다음 구조를 따른다:
  1. 스킬 이름 및 한 줄 설명
  2. 사용법 (슬래시 커맨드)
  3. 동작 방식 (단계별)
  4. 출력 형식 예시
- 하나의 스킬은 하나의 명확한 목적만 수행한다

### 스킬 품질 기준

- 사용자 입력이 모호할 경우 스킬 스스로 명확화 질문을 해야 한다
- 출력은 사용자가 즉시 활용할 수 있는 형식이어야 한다
- 정량 평가가 포함된 스킬은 반드시 수치 기준(과락선 등)을 명시해야 한다

## 향후 추가 예정 스킬 (아이디어)

- 프롬프트 유형 분류기: 주어진 프롬프트가 어떤 유형(지시형, 질문형, 역할부여형 등)인지 분류
- 프롬프트 비교 평가기: 두 버전의 프롬프트 중 어느 쪽이 더 효과적인지 비교 분석
- AI 편향 점검기: AI 결과물에 특정 편향이 포함되어 있는지 점검
