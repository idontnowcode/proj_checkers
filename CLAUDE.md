# proj_checkers

AI를 활용하는 사용자가 AI를 목적에 맞게, 효과적으로 다루고 있는지 점검하기 위한 스킬 모음 프로젝트.

## 프로젝트 목적

사용자가 AI에게 내리는 지시(프롬프트)의 품질을 높이고, AI의 결과물을 객관적으로 평가할 수 있는 도구를 만든다.

구체적으로:
- 프롬프트가 명확하고 구체적으로 작성되었는지 검토·개선
- AI 결과물이 프롬프트의 요구사항을 얼마나 잘 충족했는지 정량 평가
- 작업 실행 전 갭 스킬을 자동으로 생성하여 스킬 생태계를 점진적으로 확장

## 표준 워크플로우

```
사용자 프롬프트 (초안)
        ↓
  /prompt-clarify          ← 프롬프트 명확화 + 스킬 추천 + 갭 스킬 탐지
        ↓
  갭 스킬 있음?
   YES ↓        NO ↓
skill-creator       ↓      ← 갭 스킬을 먼저 생성 (task-orchestrator가 자동 수행)
        ↓           ↓
   /task-orchestrator       ← 스킬 조율·실행·품질검증 자동화
        ↓
   /eval-rubric             ← 최종 결과물 품질 검증 (task-orchestrator 내부 수행)
        ↓
   사용자 최종 승인
```

## 디렉토리 구조

```
proj_checkers/
└── .claude/
    └── skills/
        ├── prompt-clarify.md     # 프롬프트 명확화 스킬
        ├── eval-rubric.md        # AI 결과물 평가 지표 생성 스킬
        ├── task-orchestrator.md  # 스킬 조율·실행 팀장 스킬
        └── skill-creator.md      # 스킬 생성·개선 스킬 (Anthropic 공식)
```

## 스킬 목록

### `/prompt-clarify` — 프롬프트 명확화 도우미

사용자가 작성한 프롬프트를 분석하고, 역질문을 통해 더 명확하고 구체적인 프롬프트로 개선한다.

- 최대 3라운드 역질문 (라운드당 질문 수는 스킬이 판단)
- 3라운드 종료 또는 충분히 구체화된 시점에 개선된 프롬프트 출력
- 멀티에이전트 필요 여부 판단 및 팀 구성도 제안
- 보유 스킬 추천 및 갭 스킬 탐지

### `/eval-rubric` — 평가 지표 생성기

완성된 프롬프트를 분석해 AI 결과물을 정량적으로 평가할 수 있는 지표를 생성한다.

- 체크리스트 (Yes/No 충족 여부)
- 점수표 (항목별 배점 + 과락 기준)
- AI 결과물이 함께 제공되면 즉시 평가 수행
- 과락 항목 발생 시 구체적인 피드백 제공

### `/task-orchestrator` — 작업 실행 팀장

구체화된 최종 프롬프트를 받아 필요한 스킬을 자율 선택·조율하고 품질 검증까지 완료한다.

- 실행 계획 수립 및 사용자 승인 후 실행
- **갭 스킬 발견 시 `skill-creator`로 자동 생성 후 본 작업 진행**
- eval-rubric 품질 검증 자동 수행 (과락 시 최대 2회 자동 재작업)
- 최종 결과물 사용자 승인

### `/skill-creator` — 스킬 생성·개선 도구 (Anthropic 공식)

새로운 스킬을 처음부터 만들거나 기존 스킬을 개선하고 성능을 정량 측정한다.

- 스킬 초안 작성 → 테스트 케이스 실행 → 정량 평가 → 반복 개선
- 트리거 설명(description) 최적화로 스킬 자동 발동 정확도 향상

## 개발 가이드라인

### 스킬 추가 원칙

- 각 스킬은 `.claude/skills/` 에 독립된 `.md` 파일로 작성
- 스킬 파일 구조:
  1. YAML 프론트매터 (`name`, `description` 필수)
  2. 스킬 이름 및 한 줄 설명
  3. 사용법 (슬래시 커맨드)
  4. 동작 방식 (단계별)
  5. 출력 형식 예시
- 하나의 스킬은 하나의 명확한 목적만 수행한다
- 신규 스킬은 `skill-creator`로 초안 작성 후 95점 이상 달성 시 정식 등록

### 스킬 품질 기준

- YAML 프론트매터의 `description`은 트리거 조건을 구체적이고 능동적으로 명시
- 사용자 입력이 모호할 경우 스킬 스스로 명확화 질문을 해야 한다
- 출력은 사용자가 즉시 활용할 수 있는 형식이어야 한다
- 정량 평가가 포함된 스킬은 반드시 수치 기준(과락선 등)을 명시해야 한다
- 엣지 케이스(모호한 입력, 오류, 예외 상황)에 대한 처리 방침을 명시해야 한다

### 갭 스킬 처리 원칙

- `prompt-clarify` 실행 시 갭 스킬이 탐지되면 결과물에 명시한다
- `task-orchestrator` 실행 시 갭 스킬이 있으면 `skill-creator`로 먼저 생성한 뒤 본 작업을 진행한다
- 생성된 갭 스킬은 95점 기준으로 iteration하여 품질을 확보한다

## 향후 추가 예정 스킬 (아이디어)

- 프롬프트 유형 분류기: 주어진 프롬프트가 어떤 유형(지시형, 질문형, 역할부여형 등)인지 분류
- 프롬프트 비교 평가기: 두 버전의 프롬프트 중 어느 쪽이 더 효과적인지 비교 분석
- AI 편향 점검기: AI 결과물에 특정 편향이 포함되어 있는지 점검
