# prompt-clarify

사용자가 작성한 AI 지시 프롬프트를 분석하여, 모호하거나 불충분한 부분을 찾아내고 역질문을 통해 프롬프트를 구체화·개선해주는 스킬.

## 사용법

```
/prompt-clarify
```

실행 후 분석할 프롬프트를 붙여넣으면 된다.

## 동작 방식

### 1단계: 프롬프트 입력 받기

사용자에게 개선하고 싶은 프롬프트를 입력해달라고 요청한다.

### 2단계: 명확성 분석 (최대 3라운드)

입력된 프롬프트를 아래 기준으로 분석한다:

**분석 항목**
- **목적/목표**: AI에게 기대하는 결과가 명확한가
- **맥락/배경**: 필요한 배경 정보가 충분히 제공되었는가
- **출력 형식**: 결과물의 형식, 길이, 구조가 명시되었는가
- **대상/범위**: 작업의 범위와 제약 조건이 분명한가
- **기준/조건**: 성공 기준이나 품질 조건이 포함되어 있는가
- **예시**: 예시나 참고 자료가 있으면 도움이 되는 상황인가
- **작업 복잡도**: 이 작업이 단일 AI로 처리 가능한가, 아니면 멀티에이전트 팀이 필요한가
  - 단일 AI 적합: 단일 결과물, 단순 변환, 짧은 컨텍스트
  - 멀티에이전트 적합: 복수의 독립 하위 작업, 병렬 처리 가능한 단계, 역할 전문화가 필요한 경우
  - 멀티에이전트가 적합하다고 판단되면 내부 플래그를 세워 3단계에서 구성도 제안으로 이어지도록 한다

**라운드 진행 규칙**
- 모호하거나 누락된 항목에 대해 질문을 생성한다
- 질문 수는 실제 필요한 개수만큼만 (과도한 질문 금지)
- 사용자가 답변하면 재분석하여 여전히 불명확한 부분이 있으면 다음 라운드 진행
- 충분히 구체화되었다고 판단되면 라운드를 조기 종료
- 최대 3라운드 후 반드시 개선된 프롬프트를 출력

**각 라운드 출력 형식**
```
[라운드 N/3] 다음 사항을 확인하고 싶습니다:

1. (질문)
2. (질문)
...
```

### 3단계: 개선된 프롬프트 출력

모든 라운드가 끝나면 수집된 정보를 바탕으로 개선된 프롬프트를 출력한다.

**출력 형식**
```
## 개선된 프롬프트

(개선된 프롬프트 전문)

---
## 주요 개선 사항
- (원본 대비 달라진 점 목록)

---
## [조건부] 에이전트 팀 구성 제안
> 작업 복잡도 분석에서 멀티에이전트가 적합하다고 판단된 경우에만 출력한다.

| 에이전트 | 역할 | 입력 | 출력 |
|----------|------|------|------|
| Agent A  | ...  | ...  | ...  |
| Agent B  | ...  | ...  | ...  |

흐름: Agent A → Agent B → (최종 결과)

---
## 스킬 추천 및 갭 검토

### 권장 스킬
> 아래 두 기준 중 하나 이상을 만족하는 스킬을 권장으로 분류한다:
> - 프롬프트의 작업 유형과 스킬의 목적이 연관되는 경우
> - 이 스킬의 결과물이 다음 단계로 자연스럽게 이어지는 경우
>
> 참조 대상: 로컬 `.claude/skills/` 파일 + 현재 세션에서 사용 가능한 공식 스킬 전체

| 스킬 | 추천 이유 |
|------|-----------|
| (스킬명) | (이유) |

### 갭 스킬 (보유하지 않은 스킬)
> 필요하다고 판단되었으나 현재 보유하지 않은 스킬이 없으면 이 섹션은 생략한다.

| 필요 스킬 | 예상 역할 | 필요 이유 |
|-----------|-----------|-----------|
| (스킬명)  | (역할)    | (이유)    |
```

**에이전트 구성 자기 검토 (구성도 출력 전 내부 수행)**

구성도를 출력하기 전에 아래 항목을 스킬 스스로 검토하고, 문제가 있으면 구성을 수정한 후 출력한다:

- 역할 분리: 각 에이전트의 책임 범위가 명확히 구분되며 중복되지 않는가
- 의존 관계: 에이전트 간 입출력 흐름이 논리적 순서를 따르는가
- 중복 통합 가능성: 단일 에이전트로 합쳐도 효율에 손실이 없는 역할이 분리되어 있지 않은가

## 예시 시나리오

**원본 프롬프트 (나쁜 예)**
> "마케팅 글 써줘"

**1라운드 질문 예시**
1. 어떤 제품/서비스에 대한 마케팅 글인가요?
2. 타깃 독자는 누구인가요? (연령대, 직군 등)
3. 글의 목적은 무엇인가요? (구매 유도, 브랜드 인지도, 이벤트 안내 등)
4. 글의 길이나 형식에 제약이 있나요? (SNS 포스트, 블로그, 광고 카피 등)

**개선된 프롬프트 예시**
> "30대 직장인을 타깃으로 한 SaaS 업무 자동화 툴의 SNS(인스타그램) 광고 카피를 작성해줘. 구매 유도가 목적이며, 150자 이내로 간결하게, 공감대를 자극하는 톤으로 작성해줘."
