# eval-rubric

AI가 수행한 작업 결과물을 정량적으로 평가하기 위한 체크리스트와 점수표(과락 기준 포함)를 생성하고, 결과물이 과락 기준에 미달할 경우 구체적인 피드백을 제공하는 스킬.

## 사용법

```
/eval-rubric
```

실행 후 평가 기준을 만들 프롬프트를 붙여넣으면 된다. AI 결과물도 함께 제공하면 즉시 평가까지 수행한다.

> **워크플로우 연계**: 프롬프트가 아직 불명확하다면 `/prompt-clarify`로 먼저 구체화한 후 이 스킬을 사용하는 것을 권장한다.

## 동작 방식

### 1단계: 입력 받기

다음 두 가지를 입력받는다:

1. **평가 대상 프롬프트** (필수): AI에게 주어진 지시문
2. **AI 결과물** (선택): 평가할 AI의 답변 또는 산출물

AI 결과물이 없으면 평가 지표만 생성하고, 있으면 평가 지표 생성 후 즉시 평가를 수행한다.

### 2단계: 프롬프트 분석

프롬프트를 분석하여 다음을 파악한다:

**작업 유형 분류**

| 유형 | 해당 조건 |
|------|-----------|
| 글쓰기 | 에세이, 카피, 보고서, 이메일 등 자연어 산출물 |
| 코딩 | 함수, 클래스, 스크립트, API 등 실행 가능한 코드 |
| 분석/요약 | 데이터·문서를 해석하거나 요약하는 작업 |
| 번역/변환 | 언어·형식·구조 변환 (JSON→CSV, 한→영 등) |
| 기획/전략 | 계획서, 로드맵, 의사결정 지원 문서 |
| 복합 | 위 유형이 2개 이상 혼합된 경우 → 유형별 항목을 각각 구성 |

**파악 항목**
- 핵심 요구사항 목록 (프롬프트에서 명시된 조건 추출)
- 형식/구조 요건 (분량, 언어, 포맷 등)
- 맥락적 제약 조건 (대상 독자, 사용 목적 등)

### 3단계: 평가 지표 생성

분석 결과를 바탕으로 아래 두 가지 산출물을 생성한다.

#### 체크리스트

프롬프트의 요구사항을 Yes/No로 확인할 수 있는 항목들.

```
## 체크리스트

### 필수 항목 (모두 충족해야 함)
- [ ] (항목)
- [ ] (항목)

### 권장 항목 (품질 향상 요소)
- [ ] (항목)
- [ ] (항목)
```

#### 점수표

각 평가 항목에 배점을 부여하고 과락 기준을 명시한다.

```
## 점수표

| 평가 항목 | 배점 | 과락 기준 | 설명 |
|-----------|------|-----------|------|
| (항목명)  | N점  | N점 미만  | (평가 기준 설명) |
| ...       | ...  | ...       | ... |

**총점**: 100점
**전체 과락 기준**: 70점 미만 (조정 가능)

### 과락 항목 안내
개별 항목 과락 시, 총점과 무관하게 전체 평가를 불합격으로 처리한다.
```

**배점 결정 원칙**
- 총점은 100점 기준
- 프롬프트에서 명시적으로 요구한 조건일수록 높은 배점 (명시 조건 > 암묵적 기대)
- 작업 유형의 핵심 품질 기준에 전체 배점의 50% 이상 할당
  - 글쓰기: 주제 부합도·논리 구조가 핵심
  - 코딩: 기능 충족·동작 정확성이 핵심
  - 분석/요약: 핵심 내용 포함·정확성이 핵심
- 형식·구조 요건은 별도 항목으로 분리하되 20점 이하로 제한
- 과락 기준은 해당 항목 배점의 60% 미만으로 설정
  - 핵심 항목(배점 25점 이상)은 70% 미만을 과락으로 설정

### 4단계: 결과물 평가 (AI 결과물이 제공된 경우)

생성된 지표로 실제 결과물을 평가한다.

#### 체크리스트 평가

```
## 체크리스트 평가 결과

### 필수 항목
- [x] (충족된 항목)
- [ ] (미충족 항목) ← **미충족**

### 권장 항목
- [x] (충족된 항목)
- [ ] (미충족 항목)
```

#### 점수 집계

```
## 점수 집계

| 평가 항목 | 배점 | 획득 점수 | 과락 여부 |
|-----------|------|-----------|-----------|
| (항목명)  | N점  | N점       | 통과/과락 |
| ...       | ...  | ...       | ...       |

**총점**: N / 100점
**최종 판정**: 합격 / 불합격
```

#### 피드백 (과락 항목이 있는 경우)

```
## 피드백

### 과락 항목 상세

#### (과락 항목명) — N점 / N점 (기준: N점 이상)
- **문제점**: (무엇이 부족한지 구체적으로)
- **개선 방향**: (어떻게 수정하면 기준을 충족할 수 있는지)
- **예시**: (가능하면 개선 예시 제공)

#### ...
```

## 평가 항목 예시 (작업 유형별)

### 글쓰기 과제
| 항목 | 배점 | 과락 기준 |
|------|------|-----------|
| 주제 부합도 | 30점 | 18점 미만 |
| 논리적 구조 | 20점 | 12점 미만 |
| 문체/톤 적합성 | 20점 | 12점 미만 |
| 분량/형식 준수 | 15점 | 9점 미만 |
| 문법·맞춤법 | 15점 | 9점 미만 |

### 코드 작성 과제
| 항목 | 배점 | 과락 기준 |
|------|------|-----------|
| 기능 요구사항 충족 | 40점 | 28점 미만 |
| 코드 동작 정확성 | 25점 | 18점 미만 |
| 코드 가독성 | 20점 | 12점 미만 |
| 에러 처리 | 15점 | 9점 미만 |

### 분석/요약 과제
| 항목 | 배점 | 과락 기준 |
|------|------|-----------|
| 핵심 내용 포함 여부 | 35점 | 25점 미만 |
| 정확성 | 30점 | 21점 미만 |
| 간결성 | 20점 | 12점 미만 |
| 형식 준수 | 15점 | 9점 미만 |
