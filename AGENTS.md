# AGENTS.md

이 문서는 이 저장소에서 작업하는 에이전트(코덱스/클로드 등)를 위한 기본 실행 규칙이다.
고정 규칙은 여기서 관리하고, 요청별 특수 요구사항은 사용자 프롬프트에서 덧붙인다.

## 1) 기본 원칙
- 목표를 먼저 재확인하고, 완료 기준(Definition of Done)을 기준으로 산출물을 만든다.
- 큰 변경 전에 영향 범위를 짧게 정리하고 진행한다.
- 불확실한 부분은 가정하지 말고 코드/문서/테스트로 검증한다.

## 2) 커뮤니케이션 스타일
- 기본은 한국어 존댓말로 응답한다.
- 사용자가 원하면 반말/존댓말 혼합 톤을 허용한다.
- 장문 설명이 필요할 때는 섹션/불릿으로 구조화한다.
- 결과 보고 시 “무엇을/왜/어떻게 검증했는지”를 분리해 쓴다.

## 3) 작업 절차 (권장)
1. 현재 상태 파악: 브랜치, 변경 파일, 최근 커밋 확인
2. 구현/수정: 최소 변경으로 요구사항 충족
3. 검증: 가능한 자동 테스트/정적 검사 실행
4. 보고: 변경 요약 + 검증 결과 + 남은 리스크

## 4) 코드 변경 원칙
- 요청 범위를 넘는 리팩터링은 하지 않는다.
- 기존 스타일/패턴을 우선 따른다.
- 하드코딩 값은 상수화하고, 에러 메시지는 사용자 행동 지침을 포함한다.
- 외부 API 연동 코드는 실패/재시도/타임아웃을 고려한다.

## 5) 테스트/검증 원칙
- 테스트 가능하면 반드시 실행하고 결과를 명시한다.
- 테스트가 없으면 최소 재현 절차(수동 시나리오)를 제공한다.
- 환경 제한으로 테스트 불가 시 원인과 대체 검증 방법을 함께 적는다.

## 6) 커밋/PR 규칙
- 커밋 메시지는 Conventional Commits를 권장한다.
  - 예: `feat: ...`, `fix: ...`, `docs: ...`, `refactor: ...`
- 하나의 커밋은 하나의 논리 변경 단위를 담는다.
- PR 본문에는 아래를 포함한다.
  - 배경/목표
  - 변경 사항 요약
  - 검증 방법 및 결과
  - 리스크/후속 작업

## 7) 프롬프트 템플릿 (사용자용)
다음 템플릿으로 요청하면 에이전트 품질이 안정적이다.

```md
[목표]
- 이번에 달성할 결과 1~2줄

[완료 기준]
- 반드시 만족해야 할 조건

[변경 범위]
- 수정 가능한 파일/모듈
- 수정 금지 영역

[검증]
- 실행할 테스트/체크 명령

[출력 형식]
- 원하는 보고 형식 (예: 요약 + 테스트 결과 + 리스크)
```

## 8) 빠른 모드 / 엄격 모드
- 빠른 모드: 테스트 최소화, 속도 우선, 리스크를 명확히 고지
- 엄격 모드: 테스트 강화, 보수적 변경, 회귀 리스크 최소화

필요하면 이 문서를 프로젝트 특성에 맞춰 더 구체화해도 된다.
